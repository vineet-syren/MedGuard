Respivara CLM / eDetail Aid Brief and Plan

Objective

Create a representative-guided CLM sequence that supports a compliant HCP conversation on eligible adult patient identification, evidence context, and safety considerations for Respivara in India. The sequence must work as a guided conversation, not as a static brochure: each screen has a defined rep purpose, evidence reference, interaction event, and validation control.

Representative Talk Track Controls

  • Representatives may use only the approved screen prompt and cannot add comparative, duration, superiority, or rapid-control language.
  • Free-text claim editing is disabled; CRM notes can capture HCP interest but cannot store patient-level information.
  • Safety screen must be reached before the call-object can be marked complete.
  • All claims must carry visible claim ID, reference ID, expiry status, and permitted-market metadata.

Readiness Criteria

The CLM asset is ready only when screen-level claims match approved language, the reference drawer opens from every claim-bearing screen, the safety screen is mandatory, the India local overlay is present, no unsupported free-text claims can be introduced, and final Veeva CRM package metadata is ready for QA and publish validation.
